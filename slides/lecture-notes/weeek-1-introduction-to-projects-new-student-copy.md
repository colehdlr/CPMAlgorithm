## Page 16

Linear (Waterfall) Life Cycle
Sequential phases: Initiation 
→ Planning → Execution → 
Closure
Advantages: clarity, easy to 
manage, good for stable 
requirements
Disadvantages: inflexible, 
late testing, costly changes

## Page 17

Iterative Life Cycle
• Repeated cycles to refine outputs 
(Agile, Scrum, etc.)
• Popular Agile methodologies: 
SCRUM, Extreme Programming (XP), 
Dynamic Systems Development 
Method (DSDM), Feature Driven 
Development (FDD), and Crystal
• Advantages: adaptability, user 
feedback, faster value delivery
• Disadvantages: less predictability, 
scope creep risks

## Page 18

Hybrid Life Cycle
• Combination of linear & 
iterative approaches
• Example: Waterfall for 
hardware, Agile for software in 
same project
• Advantages: balanced 
structure + flexibility
• Disadvantages: complexity in 
governance

## Page 25

Mendelow’s Matrix
• Used to:
• To prioritise stakeholder engagement and 
communication strategies.
• To ensure that the needs and concerns of key 
stakeholders are addressed effectively.
• Mendelow (1991) categorises stakeholders 
based on two areas:
• Power: The ability to influence an organisation
strategy or project resources
• Interest: How interested they are in the 
organisation or project succeeding
